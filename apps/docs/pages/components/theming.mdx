# Theming

`@orderly.network/react` 提供了一套默认的主题, 同时也支持自定义主题，你可以参考下边的内容进行自定义。

## Deafult theme

### Colors

在`@orderly.network/react`中用到的颜色如下:

- 除了`base`之外每个色系包含3个颜色:
  - `defalut`: The defalut shade of the color
  - `light`: A lighter shade of defalut
  - `darken`: A darker shade of defalut

- `base-contrast` 定义的是文本的颜色，是以透明度来区分文字的强弱。

`@orderly.network/react`中用到的字体: Manrope

### 样式覆盖

如果在开发中碰到通过自定义 CSS 变量不能修改样式的情况可以使用样式覆盖的方式来解决，`@orderly.network/react`组件提供以下两种方式来覆盖组件的默认样式

#### class

通过重写某个 class 的定义来覆盖某一类组件的样式，例如我们想要修改`button`组件的样式，可以通过以`class`选择器选择元素, 例如:

修改所有`button`组件的样式:

```css
/* 
general buttons
*/
.orderly-button {
  background-color: #000;
  color: #fff;
}
/* 
primary buttons
*/
.orderly-button-primary {
  background-color: #000;
  color: #fff;
}
```

修改`button`组件`disabled`状态的样式:

```css
.orderly-button-disabled {
  background-color: rgba(0, 0, 0, 0.3);
  color: #999;
}
```

#### id

如果想要修改某个特定组件的样式，可以通过 id 选择器来定义组件的样式，如比想要修改创建订单按钮的样式:

```css
#orderly-order-entry-confirm-button {
  background-color: #000;
  color: #fff;
}
```

**支持的 id 如下:**

| :--------------------------------------- | :------------: | ---------------------: |

| orderly-pending-cell-edit-button         | peding orders  |   修改进行中的订单按钮 |
| orderly-pending-cell-cancel-button       | peding orders  |   取消进行中的订单按钮 |
