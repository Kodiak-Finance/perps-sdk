## 样式覆盖

如果在开发中碰到通过自定义CSS 变量不能修改样式的情况可以使用样式覆盖的方式来解决，`@orderly.network/react`组件提供以下两种方式来覆盖样式


### className

通过重写某个class的定义来覆盖某一类组件的样式，例如我们想要修改`button`组件的样式，可以通过以`class`选择器选择元素, 比如:

修改所有`button`组件的样式

```css
/* 
general buttons
*/
.orderly-button {
  background-color: #000;
  color: #fff;
}
/* 
primary buttons
*/
.orderly-button-primary {
  background-color: #000;
  color: #fff;
}
```

修改`button`组件`disabled`状态的样式

```css
.orderly-button-disabled {
  background-color: rgba(0,0,0,0.3);
  color: #999;
}

```

### id

如果想要修改某个特定组件的样式，可以通过id选择器来定义组件的样式，如比想要修改创建订单按钮的样式:

```css
#orderly-order-entry-confirm-button{
    background-color: #000;
    color: #fff;
}
```
